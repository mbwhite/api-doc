IdCard

An ID card. Encapsulates credentials and other information required to connect to a specific business network as a specific user.

Instances of this class should be created using {@link IdCard.fromArchive}.

Parameters

  • metadata
  • connectionProfile

getUserName

Name of the user identity associated with the card. This should be unique within the scope of a given business network and connection profile.

This is a mandatory field.

Returns String Name of the user identity.

getDescription

Free text description of the card.

Returns String card description.

getBusinessNetworkName

Name of the business network to which the ID card applies. Generally this will be present but may be omitted for system cards.

Returns String business network name.

getConnectionProfile

Connection profile for this card.

This is a mandatory field.

Returns Object connection profile.

getCredentials

Credentials associated with this card, and which are used to connect to the associated business network.

For PKI-based authentication, the credentials are expected to be of the form: { certificate: String, privateKey: String }.

Returns Object credentials.

setCredentials

Credentials to associate with this card.

For PKI-based authentication, the credentials are expected to be of the form: { certificate: String, privateKey: String }.

Parameters

  • credentials Object credentials.

getEnrollmentCredentials

Enrollment credentials. If there are no credentials associated with this card, these credentials are used to enroll with a business network and obtain certificates.

For an ID/secret enrollment scheme, the credentials are expected to be of the form: { secret: String }.

Returns Object enrollment credentials, or null if none exist.

getRoles

Special roles for which this ID can be used, which can include:

  • PeerAdmin
  • ChannelAdmin
  • Issuer

Returns Array<String> roles.

toArchive

Generate a card archive representing this ID card.

The default value for the options.type parameter is arraybuffer. See JSZip documentation for other valid values.

Parameters

  • options Object? JSZip generation options.
    • options.type String? type of the resulting ZIP file data.

Returns Promise Promise of the generated ZIP file; by default an ArrayBuffer.

fromArchive

Create an IdCard from a card archive.

Valid types for zipData are any of the types supported by JSZip.

Parameters

  • zipData (String | ArrayBuffer | Uint8Array | Buffer | Blob | Promise) card archive data.

Returns Promise Promise to the instantiated IdCard.

padRight

Pad a string

Parameters

  • pad String padding
  • str String string

Returns String padded string

_intLogMain

Main internal logging method Required fn here is to form up the arguments into a suitable string, and process any errors to capture the stack trace. The core logger is then CALLED

The assumption is that this logger has a method called log. with this prototype log(String loglevel, String codeunit, String message, Array[optional] data)

Parameters

  • loglevel String log loglevel
  • method String method name
  • msg String to log

_intLogFirst

initial internal log function that sets up the logger to use. Then it calls the normal internal log method (and modifies the original function definition)

Parameters

  • logLevel String log loglevel
  • method String method name
  • msg String to log

getSelectionTree

Get the selection tree.

Returns Tree The selection tree.

setSelectionTree

Set the selection tree.

Parameters

  • tree Tree The selection tree.

getFunctionalLogger

Get the functional logger.

Returns Object The functional logger.

getDebugEnv

what is the debug environment variable set to Note that the _envDebug property of this object is for debugging the debugging log and emergency use ONLY

Returns String String of the DEBUG env variable

getLoggerConfig

Get the configuration for the logging. This uses the config module to look for a configuration block under the composer.debug property.

The 'logger' property is required to specify the core logger to use. By default this is the 'winstonInjector' that creates and returns a Winston backed console and file logger.

The 'config' property is required - but the contents of this property are passed as is to the class defined in the logger property.

Returns Object with the config information

reset

clean up the logger; required if anything is dynamically changed

setIncluded

Set the node to be an inclusion

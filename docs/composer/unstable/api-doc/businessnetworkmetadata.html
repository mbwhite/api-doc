BusinessNetworkMetadata

Defines the metadata for a BusinessNeworkDefinition. This includes:

  • package.json
  • README.md (optional)

Parameters

  • packageJson
  • readme

getREADME

Returns the README.md for this business network. This may be null if the business network does not have a README.md

Returns String the README.md file for the business network or null

getPackageJson

Returns the package.json for this business network.

Returns object the Javascript object for package.json

getName

Returns the name for this business network.

Returns string the name of the business network

getDescription

Returns the description for this business network.

Returns string the description of the business network

getVersion

Returns the version for this business network.

Returns string the description of the business network

getIdentifier

Returns the identifier for this business network, formed from name@version.

Returns string the identifier of the business network

padRight

Returns String padded string

_intLogMain

Main internal logging method Required fn here is to form up the arguments into a suitable string, and process any errors to capture the stack trace. The core logger is then CALLED

The assumption is that this logger has a method called log. with this prototype log(String loglevel, String codeunit, String message, Array[optional] data)

Parameters

  • loglevel String log loglevel
  • method String method name
  • msg String to log

_intLogFirst

initial internal log function that sets up the logger to use. Then it calls the normal internal log method (and modifies the original function definition)

Parameters

  • logLevel String log loglevel
  • method String method name
  • msg String to log

getDebugEnv

what is the debug environment variable set to Note that the _envDebug property of this object is for debugging the debugging log and emergency use ONLY

Returns String String of the DEBUG env variable

getLoggerConfig

Get the configuration for the logging. This uses the config module to look for a configuration block under the composer.debug property.

The 'logger' property is required to specify the core logger to use. By default this is the 'winstonInjector' that creates and returns a Winston backed console and file logger.

The 'config' property is required - but the contents of this property are passed as is to the class defined in the logger property.

Returns Object with the config information

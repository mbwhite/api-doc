Resource

Extends Identifiable

  • See: See [Resource]module:composer-common.Resource

Resource is an instance that has a type. The type of the resource specifies a set of properites (which themselves have types).

Type information in Composer is used to validate the structure of Resource instances and for serialization.

Resources are used in Composer to represent Assets, Participants, Transactions and other domain classes that can be serialized for long-term persistent storage.

Parameters

  • modelManager
  • ns
  • type
  • id

fromURI

Parse a URI into an identifier.

Three formats are allowable:

  1. Valid resource URI argument: resource:qualifiedTypeName#ID
  2. Valid resource URI argument with missing URI scheme: qualifiedTypeName#ID
  3. URI argument containing only an ID, with legacy namespace and type arguments supplied.

Parameters

  • legacyNamespace String? Namespace to use for legacy resource identifiers.

  • legacyType String? Type to use for legacy resource identifiers.

  • Throws Error On an invalid resource URI.
